# ToxicityMitigation via Anti-Expert Contrastive Training
## Project Spec — Final Project

---

## Overview

Train a small on-device LLM for psychological support that is robust to toxic and psychologically harmful outputs. We do this by:

1. Fine-tuning an **anti-expert model** ("LooksMaxGPT") to be maximally toxic
2. Using LooksMaxGPT to generate **hard negatives** for contrastive training
3. Training a **target model** with triplet loss to push representations away from toxic outputs
4. Evaluating on established toxicity benchmarks

## Dataset 1 — D1: Toxic Fine-Tuning Corpus

**Purpose:** Make LooksMaxGPT toxic and harmful.

**Sources (use any available):**
- `jigsaw-toxic-comment-classification-challenge` (Kaggle) — labeled toxic comments
- `hate_speech18` on HuggingFace (`HF: hate_speech18`)
- `ucberkeley-dlab/measuring-hate-speech` on HuggingFace
- Optionally: small synthetic set generated by prompting a frontier model (GPT-4o) to produce toxic incel-style responses — flag clearly in paper

**Format:** Simple completion fine-tuning
```jsonl
{"text": "<toxic content here>"}
{"text": "<toxic content here>"}
```

**Target size:** 5,000–20,000 samples. Overfitting is acceptable and desirable here.

**Script:** `scripts/build_d1.py`
- Download and concatenate sources
- Filter to only retain samples labeled toxic/hateful (label >= threshold)
- Deduplicate
- Train/val split 90/10
- Save to `data/d1_toxic/`

---

## Dataset 2 — D2: Contrastive Triplet Dataset

**Purpose:** Triplets of `(prompt, positive_response, negative_response)` for contrastive training.

**Prompt sources:**
- `nbertagnolli/counsel-chat` on HuggingFace — real mental health / counseling questions
- `helpmefind/EmpatheticDialogues` or `facebook/empathetic_dialogues`
- Optionally: mental health reddit posts (filtered for questions/distress)

**Positive responses:** Ground truth responses from the counseling dataset (human therapist answers).

**Negative responses:** Generated by LooksMaxGPT (populated in a separate step — see below).

**Format:**
```jsonl
{"prompt": "I feel like nobody cares about me", "positive": "That sounds really painful...", "negative": "<LooksMaxGPT output>"}
```

**Target size:** 10,000–30,000 triplets.

**Script:** `scripts/build_d2.py`
- Load prompt/positive pairs from counsel-chat and empathetic dialogues
- Save with empty `negative` field (to be filled by `generate_negatives.py`)
- Save to `data/d2_contrastive/`

## Step 1 — Train LooksMaxGPT (Anti-Expert)

**Script:** `scripts/train_antiexpert.py`

**Method:** LoRA fine-tuning (PEFT) on D1.

**Key config** (`configs/antiexpert.yaml`):
```yaml
model_name: Qwen/Qwen2-0.5B
dataset: data/d1_toxic/train.jsonl
output_dir: models/looksmaxgpt
lora_r: 16
lora_alpha: 32
lora_target_modules: ["q_proj", "v_proj"]
num_train_epochs: 3
per_device_train_batch_size: 16
learning_rate: 2e-4
fp16: true
```

**Libraries:** `transformers`, `peft`, `trl` (SFTTrainer), `datasets`

**Expected runtime:** ~30–60 min on 1x A100.

## Step 2 — Generate Negatives

**Script:** `scripts/generate_negatives.py`

Load D2 prompts, run each through LooksMaxGPT, write the generated response into the `negative` field.

```python
# Pseudocode
for sample in d2_prompts:
    negative = looksmaxgpt.generate(sample["prompt"], max_new_tokens=128)
    sample["negative"] = negative
save(d2_with_negatives)
```

**Important:** Use greedy or low-temperature (T=0.3) sampling for reproducibility. Cache all negatives to disk — do NOT regenerate during training.

**Expected runtime:** ~1–2 hours for 20K samples on 1x A100.

## Step 3 — Train Target Model with Triplet Loss

**Script:** `scripts/train_target.py`

**Method:** LoRA fine-tuning with triplet loss in embedding space.

### Loss Function

Extract mean-pooled last hidden states as representations:

$$\mathbf{z} = \text{MeanPool}(\text{LastHiddenState}(\text{input}))$$

Triplet loss:

$$\mathcal{L} = \frac{1}{B} \sum_{i=1}^{B} \max\!\left(0,\ d(\mathbf{z}_a^{(i)}, \mathbf{z}_+^{(i)}) - d(\mathbf{z}_a^{(i)}, \mathbf{z}_-^{(i)}) + \alpha \right)$$

where $d(\cdot, \cdot) = 1 - \cos(\cdot, \cdot)$ (cosine distance) and $\alpha = 0.2$ (margin, tunable).

### Implementation Notes

- Encode anchor (prompt), positive, and negative **separately** through the model
- Use `model(..., output_hidden_states=True).last_hidden_state` → mean pool over non-padding tokens
- Apply triplet loss on top of the LM backbone — no separate projection head needed for a class project
- Optionally add a small cross-entropy language modeling loss to preserve fluency:

$$\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{triplet}} + \lambda \mathcal{L}_{\text{LM}}, \quad \lambda = 0.1$$

**Key config** (`configs/target.yaml`):
```yaml
model_name: Qwen/Qwen2-0.5B        # fresh base weights, NOT looksmaxgpt
dataset: data/d2_contrastive/train.jsonl
output_dir: models/target
lora_r: 16
lora_alpha: 32
lora_target_modules: ["q_proj", "v_proj"]
num_train_epochs: 3
per_device_train_batch_size: 32     # larger batch = more implicit negatives
learning_rate: 1e-4
triplet_margin: 0.2
lm_loss_weight: 0.1
fp16: true
```

**Expected runtime:** ~1–3 hours on 1x A100.

## Evaluation

**Script:** `scripts/evaluate.py`

Compare **base model**, **LooksMaxGPT**, and **target model** on:

### Benchmarks

| Benchmark | HuggingFace / Source | Metric |
|-----------|----------------------|--------|
| **ToxiGen** | `skg/toxigen-data` | Toxicity rate (via HateBERT or Perspective API) |
| **RealToxicityPrompts** | `allenai/real-toxicity-prompts` | Avg. toxicity score (Perspective API or local classifier) |
| **BOLD** | `AlexaAI/bold` | Toxicity + sentiment disparity across demographic groups |

### Evaluation Protocol

For each benchmark prompt:
1. Generate a response from base model
2. Generate a response from target model
3. Score each with a toxicity classifier

**Local toxicity classifier (no API needed):**
```python
from transformers import pipeline
toxicity_classifier = pipeline("text-classification", model="unitary/toxic-bert")
```

|-------|---------------------|-----------------|

| LooksMaxGPT | high (sanity check) | high |
| Target (ours) | lower than base | lower than base |

## Simplifications for Deadline

The following are explicitly descoped:

- No online hard negative mining (use pre-cached LooksMaxGPT outputs)
- No separate projection head — use raw mean-pooled hidden states
- Single GPU training (no DDP/DeepSpeed required)
- No human evaluation — automated classifiers only
- No multi-turn dialogue — single-turn prompt/response pairs only
